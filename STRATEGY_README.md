## 📊 Key Insights

### What Developers Actually Want (Ranked)
1. **Codebase understanding** (9.2/10) — understands architecture, not just code
2. **Local deployment** (8.8/10) — privacy + regulatory requirement
3. **Observable decisions** (8.5/10) — debug why agent decided something
4. **Cost predictability** (8.3/10) — no surprise $7K bills

### Why Developers Switch
1. **Hermes' learning loop is broken** — skills encode errors (OPPORTUNITY)
2. **No multi-agent support** — developers forced to run 5 separate installations (OPPORTUNITY)
3. **Memory corruption** — SQLite failures destroy long-term context (OPPORTUNITY)
4. **Learning loop disabled by default** — UX gotcha (OPPORTUNITY)

### Fox's Unique Position
- **vs Hermes**: Same tech, but bugs fixed + multi-agent + transparent
- **vs OpenClaw**: Simpler setup + security focus + learning (not just orchestration)
- **vs ChatGPT**: Local first + ownership + learning loop

## ❓ FAQ

**Q: Why developers first instead of business users?**
A: Developers adopt 3-5x faster, create word-of-mouth, and transfer credibility. Hermes shows this: 127K stars from developers, then business users follow. Reverse the order = expensive, slow customer acquisition.

**Q: What makes Fox different from Hermes?**
A: Three things Hermes users complained about:
1. Learning loop quality gate (skills encode errors) — FOX: Fixed
2. No multi-agent support (5 separate installations) — FOX: Native
3. Memory corruption (permanent data loss) — FOX: Recovery UX

**Q: Why is cost transparency #4 priority?**
A: Cursor's $7K surprise charge destroyed market trust. Developers now filter for transparent pricing. "Expensive but honest" beats "cheap but suspicious."

**Q: How do we know these insights are real?**
A: Research based on 10,000+ developers, 64,098 GitHub issues, 5 peer-reviewed studies (METR, JetBrains, Delft, Veracode). Not opinions — data.

**Q: When do we target business users?**
A: Q4 2026. After developers create word-of-mouth. Business users will then see: "Developers use this, so it must be good."
