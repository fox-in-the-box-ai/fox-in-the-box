# 06 — CI Pipeline

## Trigger

Same as existing release flow — on `git tag v*`. The `.deb` build is an additional parallel job alongside the existing Docker image build.

## New jobs in release.yml

```yaml
jobs:
  # ── existing jobs (unchanged) ──────────────────────────────────────────────
  build-docker: ...
  build-electron: ...

  # ── new: build .deb for amd64 and arm64 ───────────────────────────────────
  build-deb:
    strategy:
      matrix:
        arch: [amd64, arm64]
    runs-on: ${{ matrix.arch == 'arm64' && 'ubuntu-22.04-arm' || 'ubuntu-22.04' }}
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: recursive

      - name: Build .deb
        run: |
          sudo apt-get install -y dpkg-dev
          bash packages/deb/build.sh ${{ matrix.arch }}

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: deb-${{ matrix.arch }}
          path: dist/*.deb

  # ── new: publish to apt repo ──────────────────────────────────────────────
  publish-apt:
    needs: build-deb
    runs-on: ubuntu-22.04
    steps:
      - uses: actions/download-artifact@v4
        with:
          pattern: deb-*
          merge-multiple: true
          path: dist/

      - name: Publish to apt.foxinthebox.ai
        env:
          GPG_PRIVATE_KEY: ${{ secrets.APT_GPG_PRIVATE_KEY }}
          GPG_KEY_ID:      ${{ secrets.APT_GPG_KEY_ID }}
          R2_ACCOUNT_ID:   ${{ secrets.R2_ACCOUNT_ID }}
          R2_ACCESS_KEY:   ${{ secrets.R2_ACCESS_KEY }}
          R2_SECRET_KEY:   ${{ secrets.R2_SECRET_KEY }}
        run: bash packages/deb/publish-apt.sh
```

## Secrets required (new)

| Secret | Description |
|--------|-------------|
| `APT_GPG_PRIVATE_KEY` | Armored GPG private key for signing apt Release files |
| `APT_GPG_KEY_ID` | Key ID / fingerprint |
| `R2_ACCOUNT_ID` | Cloudflare account ID |
| `R2_ACCESS_KEY` | R2 access key (read+write on foxinthebox-apt bucket) |
| `R2_SECRET_KEY` | R2 secret key |

## publish-apt.sh

Extracted as a standalone script (not inline YAML) for local testing:

```bash
#!/usr/bin/env bash
set -euo pipefail

# Install tools
sudo apt-get install -y -q reprepro gpg
curl -fsSL https://rclone.org/install.sh | sudo bash

# Import GPG key
echo "$GPG_PRIVATE_KEY" | gpg --batch --import

# Configure rclone for R2
mkdir -p ~/.config/rclone
cat > ~/.config/rclone/rclone.conf << EOF
[r2]
type = s3
provider = Cloudflare
access_key_id = $R2_ACCESS_KEY
secret_access_key = $R2_SECRET_KEY
endpoint = https://${R2_ACCOUNT_ID}.r2.cloudflarestorage.com
EOF

# Pull current apt repo state
mkdir -p ./apt-repo
rclone sync r2:foxinthebox-apt ./apt-repo --quiet

# Ensure reprepro conf exists (bootstrap on first run)
mkdir -p ./apt-repo/conf
if [ ! -f ./apt-repo/conf/distributions ]; then
    cat > ./apt-repo/conf/distributions << EOF2
Origin: Fox in the Box
Label: Fox in the Box
Codename: stable
Architectures: amd64 arm64
Components: main
SignWith: $GPG_KEY_ID
EOF2
fi

# Add new .deb files
for deb in dist/*.deb; do
    arch=$(dpkg --field "$deb" Architecture)
    reprepro -b ./apt-repo includedeb stable "$deb"
    echo "Added: $deb (arch=$arch)"
done

# Sync back to R2
rclone sync ./apt-repo r2:foxinthebox-apt --quiet

echo "Published successfully to apt.foxinthebox.ai"
```

## Local testing

```bash
# Build the .deb locally
bash packages/deb/build.sh amd64

# Test install in a clean Ubuntu 22.04 container
docker run --rm -it \
  -v $(pwd)/dist:/debs \
  ubuntu:22.04 bash -c "
    apt-get update -q &&
    apt-get install -y -q /debs/foxinthebox_*.deb &&
    sleep 10 &&
    curl -f http://localhost:8787/health &&
    echo 'PASS: health check ok'
  "
```

This is the acceptance test for the .deb. Add to CI as a smoke test job gated after `build-deb` and before `publish-apt`.
