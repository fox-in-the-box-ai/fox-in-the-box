## Summary

Set up a **pnpm workspace monorepo** with the correct directory layout, root `package.json`, `pnpm-workspace.yaml`, two git submodules pointing at forked repos, and stub files for every package that will be developed in later tasks.

No real application code is written here — this task is purely structural scaffolding.

## Step-by-Step Instructions

### 1. Create directory skeleton

```bash
cd /home/ubuntu/workspace/fox-in-the-box

mkdir -p .github/workflows
mkdir -p forks
mkdir -p packages/integration/default-configs
mkdir -p packages/electron/src
mkdir -p packages/scripts/dev
mkdir -p tests/integration tests/electron tests/container
```

### 2. Add git submodules

```bash
cd /home/ubuntu/workspace/fox-in-the-box

git submodule add -b main \
  https://github.com/fox-in-the-box-ai/hermes-agent.git \
  forks/hermes-agent

git submodule add -b main \
  https://github.com/fox-in-the-box-ai/hermes-webui.git \
  forks/hermes-webui
```

This creates `.gitmodules` automatically.

### 3. Create `pnpm-workspace.yaml`

```yaml
# pnpm-workspace.yaml
packages:
  - "packages/*"
```

> **Note:** `forks/` is intentionally excluded from the pnpm workspace — the submodules are consumed via Docker `COPY`/build steps, not as pnpm packages.

### 4. Create root `package.json`

```json

    "node": ">=20",
    "pnpm": ">=9"
  },
  "packageManager": "pnpm@9.0.0",
  "scripts": {
    "install:all": "pnpm install --recursive",
```

## Notes & Decisions

| Decision | Rationale |
|---|---|
| pnpm@9 | Lockfile v9, workspace protocol support, fast installs |
| Node 20 | Current LTS; matches Electron 29+ requirements |
| `forks/` excluded from pnpm workspace | Submodule repos have their own dependency management; they are consumed as file trees, not pnpm packages |
| No real deps in `packages/electron` yet | Electron version is pinned in task 06 once the container baseline is settled |
| `default-configs/` uses placeholder values | Prevents secrets from ever touching the repo; real values injected at runtime |
| git submodule branch pinned to `main` | Simplest tracking strategy; a future task can add a Renovate rule to pin to tags |
