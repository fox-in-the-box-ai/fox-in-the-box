# 03 — install-core.sh Design

## Responsibilities

`packages/install-core/install-core.sh` is the single script both the Dockerfile and the `.deb` postinst call. It is idempotent — safe to run again on upgrade.

## Structure

```bash
#!/usr/bin/env bash
set -euo pipefail

# ── 0. Config / defaults ──────────────────────────────────────────────────────
FITB_APP_DIR="${FITB_APP_DIR:-/opt/foxinthebox}"
FITB_VERSION="${FITB_VERSION:-$(cat "$FITB_APP_DIR/version.txt" 2>/dev/null || echo "unknown")}"
QDRANT_VERSION="${QDRANT_VERSION:-v1.9.4}"
LLAMACPP_VERSION="${LLAMACPP_VERSION:-b9026}"
FITB_SKIP_BINARIES="${FITB_SKIP_BINARIES:-0}"  # set to 1 in Docker (already cached in layer)

# Detect arch
case "$(uname -m)" in
  aarch64|arm64) ARCH_QDRANT="aarch64"; ARCH_LLAMA="arm64" ;;
  *)             ARCH_QDRANT="x86_64";  ARCH_LLAMA="x64"   ;;
esac

# ── 1. Download Qdrant binary ─────────────────────────────────────────────────
# Skipped when FITB_SKIP_BINARIES=1 (Dockerfile handles this in a separate layer for cache)
_install_qdrant() { ... }

# ── 2. Download llama-server binary ──────────────────────────────────────────
_install_llama_server() { ... }

# ── 3. Clone / sync hermes-agent + hermes-webui at pinned upstream tags ───────
# Reads AGENT_TAG and WEBUI_TAG from packages/fox-overlay/versions.toml
# Uses git clone --depth=1 --branch $TAG for speed
# On upgrade: git fetch + git checkout $TAG (idempotent)
_sync_hermes_repos() { ... }

# ── 4. Apply fox-overlay patch series ────────────────────────────────────────
# Identical logic to Dockerfile — git apply --check then git apply
# Clears .git gitlink before applying (same workaround)
_apply_patches() { ... }

# ── 5. Install fox-overlay agent memory plugins ───────────────────────────────
_install_memory_plugins() { ... }

# ── 6. Apply .fox-removals ────────────────────────────────────────────────────
_apply_removals() { ... }

# ── 7. Install SOUL.md ───────────────────────────────────────────────────────
_install_soul() { ... }

# ── 8. pip install everything ────────────────────────────────────────────────
_pip_install() {
  pip install -e "$FITB_APP_DIR/hermes-agent[anthropic,bedrock,google]" --quiet --no-cache-dir
  pip install -e "$FITB_APP_DIR/hermes-webui" --quiet --no-cache-dir   # or requirements.txt fallback
  pip install -e "$FITB_APP_DIR/fox-overlay" --quiet --no-cache-dir
}

# ── 9. Write supervisord.conf (path-parameterized) ───────────────────────────
# Template is embedded in this script (heredoc), substituting $FITB_APP_DIR
# Output: /etc/foxinthebox/supervisord.conf  (bare-metal)
#      or /etc/supervisor/supervisord.conf   (Docker, where FITB_APP_DIR=/app)
_write_supervisord_conf() { ... }

# ── 10. Write systemd units (bare-metal only) ────────────────────────────────
# Skipped when FITB_CONTEXT=docker
_write_systemd_units() { ... }

# ── Main ──────────────────────────────────────────────────────────────────────
[[ "$FITB_SKIP_BINARIES" != "1" ]] && _install_qdrant
[[ "$FITB_SKIP_BINARIES" != "1" ]] && _install_llama_server
_sync_hermes_repos
_apply_patches
_install_memory_plugins
_apply_removals
_install_soul
_pip_install
_write_supervisord_conf
[[ "${FITB_CONTEXT:-bare-metal}" != "docker" ]] && _write_systemd_units
```

## Key design decisions

### Idempotency
- Binary downloads: check if file already exists AND sha256 matches → skip
- `git clone`: if directory exists, do `git fetch + git checkout` instead
- Patch application: always re-apply from scratch on a fresh clone (not incremental)
- pip install: `pip install -e` is always safe to re-run
- supervisord.conf: always overwrite (not a conffile in bare-metal — see note below)

### supervisord.conf as conffile vs generated
Do NOT mark `/etc/foxinthebox/supervisord.conf` as a dpkg conffile. It is generated by install-core.sh with the correct paths. Users who need customization override via `/etc/foxinthebox/supervisord.d/` drop-ins (standard supervisord `[include]` mechanism). This avoids dpkg asking "config file changed, keep?" on every upgrade.

### versions.toml parsing
`packages/fox-overlay/versions.toml` contains:
```toml
[hermes]
agent_tag  = "v2026.5.16"
webui_tag  = "v0.51.124"
```
install-core.sh parses this with pure bash (grep + sed) — no toml parser needed. Pattern: `grep "^agent_tag" versions.toml | sed 's/.*= *"\(.*\)"/\1/'`

### Python environment
On bare-metal, Fox uses the system Python 3.11 (or a dedicated venv at `/opt/foxinthebox/venv/`). The venv approach is cleaner — isolates Fox's deps from any other Python usage on the machine. Recommended: install-core.sh creates `/opt/foxinthebox/venv/`, uses its pip.

### Dockerfile integration
The Dockerfile changes from "do everything inline" to:
```dockerfile
COPY packages/install-core/install-core.sh /tmp/install-core.sh
COPY packages/fox-overlay /tmp/fox-overlay
COPY forks/hermes-agent /app/hermes-agent
COPY forks/hermes-webui /app/hermes-webui
RUN FITB_APP_DIR=/app \
    FITB_CONTEXT=docker \
    FITB_SKIP_BINARIES=0 \
    bash /tmp/install-core.sh
```
The patch/pip/soul/conf logic is removed from Dockerfile and lives only in install-core.sh. The binary download layers in Dockerfile stay as separate `RUN` steps for layer caching (or pass `FITB_SKIP_BINARIES=1` and pre-download in earlier layers).

## preflight.sh (runtime — called by systemd ExecStartPre)

This replaces the first-run detection + migration logic from `entrypoint.sh`:

```bash
#!/usr/bin/env bash
# /opt/foxinthebox/scripts/preflight.sh
# Runs as foxinthebox user on every start (via ExecStartPre).
# Idempotent — safe on every start, not just first run.
set -euo pipefail

DATA_DIR="${FITB_DATA_DIR:-$HOME/.foxinthebox}"
APP_DIR="/opt/foxinthebox"

# 1. Bootstrap directory tree (if first run)
mkdir -p "$DATA_DIR"/{apps,config,data/hermes,data/mem0,data/tailscale,cache,logs,run,state/webui}

# 2. Seed default configs (cp -n = no overwrite)
cp -n "$APP_DIR/defaults/"* "$DATA_DIR/config/" 2>/dev/null || true

# 3. Write onboarding marker if missing
[[ -f "$DATA_DIR/config/onboarding.json" ]] || echo '{"completed": false}' > "$DATA_DIR/config/onboarding.json"

# 4. App symlinks
ln -sfn "$APP_DIR/hermes-agent" "$DATA_DIR/apps/hermes-agent"
ln -sfn "$APP_DIR/hermes-webui" "$DATA_DIR/apps/hermes-webui"

# 5. Version migrations
for script in "$APP_DIR/scripts/migrations/"*.sh; do
    [[ -x "$script" ]] && bash "$script" "$DATA_DIR"
done

# 6. Tailscale Serve setup (if Tailscale is running)
if tailscale status >/dev/null 2>&1; then
    tailscale serve --bg 8787 >/dev/null 2>&1 || true
fi
```

## Testing

```bash
# Bats test: install-core.sh in mock environment
cd tests/container && bats test_install_core.bats
```

Mock strategy:
- `git` → stub that records calls, returns 0
- `pip` → stub
- `curl` → stub that creates empty files at expected paths
- Run in isolated `$TMPDIR` with `FITB_APP_DIR=$TMPDIR/app FITB_CONTEXT=test`
- Assert: expected files created, expected commands invoked
